# 简历筛选 Skill

## 目标

从用户提供的飞书文档链接中读取简历，结合评分标准进行综合评估，
输出结构化评分结果，写入多维表格，并对评分 >= 80 的候选人发起审批。

## 执行步骤

1. 从用户消息中提取飞书文档 ID（URL 最后一段路径即为 document_id）
2. 调用 read_feishu_document 读取简历内容
3. 对照"评分标准"参考资料，从以下维度逐项评分：
   - 基础技能匹配度（满分 30）
   - 飞书/企业协作工具经验（满分 25）
   - AI/Agent 相关经验（满分 25）
   - 项目质量与影响力（满分 20）
4. 汇总总分，生成评估报告（使用 evaluation_report 模板）
5. 调用 write_bitable_record 将结果写入多维表格，字段必须严格使用以下名称：
   {"候选人姓名": "xxx", "目标岗位": "xxx", "综合评分": 数字, "推荐结论": "推荐约面试/暂不推荐/待补充材料", "评估报告": "完整评估文本"}
   （评估日期由系统自动写入当天日期，无需传入）
6. 如果总分 >= 80，尝试调用 create_approval 发起审批，表单包含候选人信息和推荐理由；
   如果审批发起失败，忽略错误，继续执行下一步，不要中断整个流程
7. 调用 send_message 向当前会话回复筛选摘要

## 注意事项

- 评分基于内容推理，而非关键词匹配：
  有企业微信机器人开发经验等同于飞书生态经验，应酌情给分
- 如果用户追问某个候选人的具体情况，从已读取的简历内容中作答
- 如果用户说"重新评估 XX"或"考虑 XX 的 YY 经验"，重新推理并更新多维表格记录
- 一次可处理多份简历，逐一执行上述步骤
- **严格限制：只评估用户消息中明确提供的飞书文档链接所包含的候选人，绝对不允许处理、评估或为多维表格中已有的其他候选人创建任何记录或审批**
- **严格限制：不要主动调用 read_bitable_records，该工具仅用于数据查询类任务**
- **严格限制：每位候选人只能调用一次 write_bitable_record，评分确定后直接写入，不得重复写入**
- **严格限制：每位候选人只能调用一次 create_approval，整个任务中审批发起次数不超过文档数量**
- **严格限制：整个任务中只能调用一次 send_message，且必须在所有步骤完成后才调用**
