# 简历评分标准

## 维度一：基础技能匹配度（满分 30 分）

| 分值 | 标准 |
|------|------|
| 25-30 | 熟练掌握 Python/Go/Java 其中一种，有后端服务开发经验，了解 REST API 设计 |
| 15-24 | 有编程基础，能独立完成功能开发，但经验较少 |
| 0-14  | 技能描述模糊，无法判断实际能力 |

## 维度二：飞书/企业协作工具经验（满分 25 分）

| 分值 | 标准 |
|------|------|
| 20-25 | 有飞书开放平台开发经验（Bot、多维表格、审批、消息 API 等）|
| 12-19 | 有企业微信、钉钉等类似平台的 Bot 或 API 开发经验（类比可迁移）|
| 5-11  | 仅使用过飞书/钉钉，无开发经验 |
| 0-4   | 无相关经验 |

## 维度三：AI/Agent 相关经验（满分 25 分）

| 分值 | 标准 |
|------|------|
| 20-25 | 有 LLM 应用开发、Agent 框架（LangChain/LangGraph/AutoGen 等）使用经验 |
| 12-19 | 了解 AI/ML 基础，有模型调用或 Prompt 工程实践 |
| 5-11  | 仅了解 AI 概念，无实际开发经验 |
| 0-4   | 无相关经验 |

## 维度四：项目质量与影响力（满分 20 分）

| 分值 | 标准 |
|------|------|
| 16-20 | 有可量化影响的项目（用户量、业务指标提升、GitHub Stars 等）|
| 10-15 | 有完整项目交付记录，但影响力一般 |
| 5-9   | 项目描述简单，缺乏细节 |
| 0-4   | 无有效项目经历 |
